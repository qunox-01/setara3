# Xariff.com Landing Page Structure

## Goal of the page

Shift Xariff from a free CSV tool impression to a **data terrain analysis service** impression.

The page should do two things well:

1. attract qualified traffic
2. keep visitors reading long enough to understand the value and move toward inquiry

---

## Core principles for the page

- Explain **what Xariff is** within the first screen
- Translate **data terrain** into familiar ML pain points immediately
- Use one clear primary action across the page
- Use visual examples early so visitors do not have to imagine the service
- Keep free tools visible, but lower on the page so they support the service instead of defining it
- Every section should answer one question only

---

## 3. What is data terrain analysis?

**Purpose:** explain the category immediately after the hero so visitors understand the service before going deeper

### Layout

Two-column section on desktop

- **Left column:** short explanation
- **Right column:** simple annotated terrain visual

Stacked layout on mobile

### Section heading

**What is data terrain analysis?**

### Copy direction

Data terrain analysis is a way of looking at a dataset as a landscape instead of just a spreadsheet.

It shows where data is dense, sparse, imbalanced, drifting, unusual, or weakly represented across classes, features, and splits.

Instead of relying only on averages or summary metrics, teams can see where the dataset is strong, where it is fragile, and where the model may need caution.

### Media placement

Use a simple visual that explains the idea fast:

- dense regions vs sparse regions
- anomaly markers
- split mismatch overlays
- strong zones vs caution zones

### Notes

- Keep this section short and easy to scan
- The goal is clarity, not methodology depth
- This section should reduce confusion around the term **data terrain** immediately

Xariff gives ML teams a structured view of how machine learning data is distributed, where it is weak, and how model behavior changes across the data terrain.

## 6. Why this matters

**Purpose:** translate features into consequences and business relevance

### Layout

Alternating text-and-visual rows or a 2-column comparison block

### Section heading

**Why data terrain analysis matters**

### Suggested content blocks

#### Block A

**Average metrics can hide dangerous weak spots**
A model can look fine overall while failing in sparse, unusual, or underrepresented regions.

#### Block B

**Train, validation, and test can disagree quietly**
Split mismatch and drift can make evaluation look more trustworthy than it really is.

#### Block C

**Rare cases often matter more than averages**
Outliers, edge cases, and long-tail examples are often where real-world failure begins.

#### Block D

**Better visibility leads to better optimization decisions**
Instead of guessing what to collect or generate next, teams can target the exact weak zones.

### Media placement

Right side or below each block:

- mini heatmaps
- mismatch overlay graphics
- anomaly flags
- confidence vs caution maps

# Recommended Button Strategy

## Primary CTA everywhere

**Book a Data Terrain Audit**

## Secondary CTA used in hero, sample section, and final CTA

**See a Sample Report**

## Tertiary / lower-priority links

- Explore Free Tools
- Talk through your use case
- Request details

### Rules

- Only one visually dominant button per section
- Avoid changing the wording of the primary CTA across the page
- Keep tool CTAs visually subordinate on the main landing page

---

# Recommended Media Strategy

## Hero media

One polished machine learning dataset analysis dashboard mockup

## Mid-page media

Small supporting visuals beside explanatory sections

## Main proof media

Annotated sample report images in the deliverables section

## Lower page media

Minimal visuals in FAQ and trust areas

### Rules

- Show product-like visuals, not abstract blobs
- Every image should teach something
- Use annotations to help visitors read the visuals quickly
- Keep screenshots clean and legible

---

# Reading Flow Logic

The visitor should move through these questions in order:

1. What is Xariff?
2. What is data terrain analysis?
3. Why should I care?
4. What exactly does it analyze?
5. What do I get from it?
6. How does it work?
7. Is this relevant to my team?
8. Can I trust it?
9. What should I do next?

---

# Mobile behavior priorities

- Keep hero copy short enough to fit before heavy scrolling
- Stack buttons vertically if needed
- Convert four-card grids into swipe or stacked cards
- Turn sample report gallery into a simple carousel
- Keep sticky CTA available but small
- Use accordion for FAQ and possibly for the “Why this matters” details

---

# What should NOT appear near the top

- Long founder story
- Too many tool cards above the fold
- Technical jargon without translation
- Multiple competing CTAs
- Generic stock photography
- Dense paragraphs explaining internal methodology too early

---

# Design tone

- Calm and technical
- High trust
- Modern but restrained
- Clear visual hierarchy
- More like a specialist machine learning dataset analysis consultancy / platform than a generic free-tool website
